# Recovery Audit & Runbook

## Automated Backups
Database backups are automatically generated by the `backup_database` management command.
- Location: `/backups/` directory in the project root.
- Format: `ecofleet_YYYYMMDD_HHMMSS.sqlite3.gz`.
- Retention Policy: Keep last 30 backups.

## Restoring from a Backup
**CRITICAL:** Restoring from a backup will permanently overwrite the current live database.

1. Locate the desired backup file in the `/backups/` folder (e.g. `ecofleet_20260626_115519.sqlite3.gz`).
2. Run the command:
   `python manage.py restore_database ecofleet_20260626_115519.sqlite3.gz`
3. The system will prompt for a manual confirmation. Type `RESTORE` to proceed.
4. A safety copy of the existing database is kept as `.safety_copy_before_restore` alongside the `.sqlite3` file, just in case.

## System Failure
If the UI or dashboard becomes unavailable:
1. Check the `logs/application.log` and `logs/errors.log`.
2. Inspect the SQLite file size (if >2GB it may cause locks/slowdowns, run `optimize_database`).
3. Check `SystemEvent` records if the admin panel is still accessible.
